# Section 9: Metrics Framework

## 9.1 North Star

The core business metric is **high-quality listings created and sustained over time**.

Listings drive trader attention, which drives fee generation.
Without listing depth, user growth metrics can be misleading.

## 9.2 KPI Layers

### Supply KPIs
- new listings per week
- active markets
- repeat listers

### Demand KPIs
- traders per active market
- fee-generating volume
- market turnover concentration

### Growth KPIs
- referral attach rate
- code-to-first-trade conversion
- code-to-first-listing conversion
- claim conversion rate (off-chain to on-chain points)
- pack conversion rate (claimed points to packs)

### Game-Layer KPIs
- packs held vs packs opened ratio
- artifact trading volume
- artifact equip rate by slot type
- median embedded-point exposure per active wallet

### Integrity KPIs
- suspicious volume incidence
- dispute rate per 1,000 users
- claim anomaly frequency
- vesting release anomaly rate
- EV deviation monitoring (expected vs realized pack output)

## 9.3 Reporting Rules

- separate organic growth from campaign imports,
- keep formula versions explicit,
- and publish phase labels (pre-beta, beta, open) to avoid false comparisons.

## 9.4 Decision Use

Metrics should inform policy, not substitute for it.
A metric that rises while market quality falls is not success.
