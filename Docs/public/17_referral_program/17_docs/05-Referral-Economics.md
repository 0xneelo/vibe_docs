# Section 5: Referral Economics

## 5.1 Why Referral Systems Get Gamed

Most abuse appears in three patterns:

1. self-referral loops,
2. code-shopping for better referee terms,
3. attribution disputes after downstream growth.

## 5.2 Uniform Referee Benefit

A key design discussed in the transcript is uniform referee benefit:

- any valid code grants the same referee baseline benefit,
- code quality depends on issuer eligibility, not discount magnitude.

This reduces "best code" arbitrage and keeps social referrals viable.

## 5.3 Issuance Gating

Unrestricted code issuance creates incentive spam.
A qualified-issuer model (activity or tier threshold, plus admin campaign grants) improves signal quality and policy control.

## 5.4 Referral Ownership Model

The model should define whether referral rights are:

- account-bound,
- NFT-portable,
- or market-scoped.

Mixing these without clear precedence leads to economic ambiguity.

## 5.5 Policy Objective

The goal is not maximum code count.
The goal is durable, fee-producing referral attribution that scales without collapsing into self-directed extraction.

## 5.6 Early-Code Scarcity and Master Cohorts

The transcript also explores an early-cohort model where initial referral code holders receive structurally stronger distribution rights during the access-gated phase.

If used, this model should define:

- exact cohort size and eligibility path,
- duration of enhanced rights,
- propagation depth limits,
- and sunset rules that prevent permanent centralization.

Scarcity can drive growth velocity, but only when paired with transparent issuance and expiry policy.
